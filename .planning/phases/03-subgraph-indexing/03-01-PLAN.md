## Wave 3: Signer Integration & Security

### Task 3.1 — Integrate Subgraph into Signer
<task>
<name>Integrate Subgraph into Signer Endpoint</name>
<objective>Use subgraph to validate sponsorship targets; remove "trust any target" fallback</objective>
<actions>
1. Update `services/signer/src/index.ts`:
   - Add SUBGRAPH_URL env var (required, fail if missing)
   - Create GraphQL client to query subgraph

2. Implement `/sponsor` endpoint validation:
   ```typescript
   // Before sponsoring, query subgraph:
   query IsValidProposal($org: ID!, $proposalId: ID!) {
     proposal(id: "$org:$proposalId") {
       id org proposer
     }
   }
   ```
   - If proposal found and org is valid → sponsor
   - If not found or subgraph unreachable → 400 error (fail closed)
   - Remove fallback to "trust any target"

3. Validate org membership:
   ```typescript
   query HasMembership($org: ID!, $user: ID!) {
     member(id: "$org:$user") { id org user }
   }
   ```
   - Reject if user not a member of org

4. Log all validation decisions for audit trail
</actions>
<verification>
- SUBGRAPH_URL required and validated at startup
- Signer rejects sponsorships for invalid proposals
- Signer rejects sponsorships for non-members
- Signer fails fast if subgraph unreachable
- No "trust any target" fallback exists
</verification>
</task>

### Task 3.2 — Add HMAC Auth to Signer Endpoint
<task>
<name>Add HMAC Authentication to Signer</name>
<objective>Secure signer `/sponsor` endpoint with HMAC before public exposure</objective>
<actions>
1. Choose auth method: HMAC (recommended for simplicity)
   - Client sends: `POST /sponsor` with body + `X-Signature: hmac-sha256(body, secret_key)`
   - Signer verifies signature using shared secret

2. Update `services/signer/src/index.ts`:
   ```typescript
   const SIGNER_SECRET = process.env.SIGNER_SECRET;
   
   app.post('/sponsor', (req, res) => {
     const signature = req.headers['x-signature'];
     const body = JSON.stringify(req.body);
     const expected = crypto
       .createHmac('sha256', SIGNER_SECRET)
       .update(body)
       .digest('hex');
     
     if (signature !== expected) return res.status(401).send('Unauthorized');
     // ... continue with sponsorship
   });
   ```

3. Add SIGNER_SECRET to `.env` (generate: `node -e "console.log(crypto.randomBytes(32).toString('hex'))"`)
4. Document in README: how to generate signature in client libraries
5. Add request logging with sanitized HMAC (no secret in logs)
</actions>
<verification>
- SIGNER_SECRET required at startup
- Requests without valid HMAC rejected with 401
- Valid HMAC requests processed normally
- Secret not logged or exposed
- README documents auth flow
</verification>
</task>
